## 2. 页面功能

### 2.1 已确认的核心职责

Legacy 酒店列表页承担国内酒店搜索结果承接页职责：

- 从酒店搜索页接收目的地城市、入住/离店日期、关键词/位置等搜索条件。
- 调用酒店列表接口分页加载酒店资源。
- 提供筛选/排序能力，帮助用户按推荐、价格、星级、区域/商圈、协议等条件缩小列表。
- 展示酒店卡片，包含图片、酒店名称、位置/评分/标签/价格/差旅政策相关信息等。
- 点击酒店卡片进入酒店详情页，并保留当前搜索条件。

### 2.2 页面结构

Legacy 页面结构如下：

- 顶部固定 Header：
  - 返回按钮。
  - `date-city_ryx` 查询条：展示入住日期、离店日期、晚数、城市、关键词占位/已选关键词。
  - 查询条交互：城市点击、日期点击、关键词点击。
- 筛选 Tab：
  - `推荐`
  - `星级/价格`
  - `位置区域`
  - `筛选`
- 列表内容：
  - 下拉刷新。
  - 首屏骨架屏。
  - 酒店列表。
  - 空态：展示 `unable_date.png` 与「暂无数据」。
  - 无限滚动：底部加载更多，文案「正在加载...」。
- 浮层：
  - 筛选面板从 Header 下方展开。
  - 展开筛选时，列表滚动禁用并展示半透明遮罩；点击遮罩关闭筛选。
- 业务提示：
  - 若公司开通「超标随心住」且当前为自订、非代理模式，列表顶部展示「贵公司已开通超标随心住，自行支付超标部分即可享受心仪的房间」，点击可打开说明弹窗。

### 2.3 页面交互

- 修改城市：
  - 打开酒店城市选择浮层。
  - 选择新城市后清空当前位置 `myPosition`。
  - 若城市发生变化，清空关键词并刷新列表。
- 修改日期：
  - 打开日历选择入住/离店日期。
  - 更新 `BeginDate`、`EndDate` 后保留当前筛选条件刷新列表。
- 修改关键词：
  - 跳转 `tmc-hotel-searchtext_ryx`。
  - 关键词返回后刷新列表。
- 返回：
  - 先关闭筛选面板，再执行页面返回。
- 点击酒店：
  - 跳转 `tmc-hotel-detail_ryx`。
  - 传参：`hotelId = item.Hotel.Id`、`hotelprice = item.AvgPrice`。

## 3. 查询与筛选

### 3.1 接口

- 查询条件接口：`TmcApiHotelUrl-Condition-Gets`
  - 入参：`cityCode`
  - 返回：品牌、设施/服务/主题、地理位置、TMC 信息等筛选元数据。
- 酒店列表接口：`TmcApiHotelUrl-Home-List`
  - Legacy 通过 `TmcHotelRyxService.getHotelList(query)` 调用。
  - 请求体由 `HotelQueryEntity`、`SearchHotelModel`、乘客、出差单和本地城市上下文共同组装。

### 3.2 列表基础查询参数

Legacy 列表请求核心字段：

| 字段 | 来源 | 说明 |
| --- | --- | --- |
| `CityCode` | 搜索模型目的地城市 | 酒店目的地城市编码 |
| `BeginDate` | 搜索模型入住日期 | 入住日期 |
| `EndDate` | 搜索模型离店日期 | 离店日期 |
| `PageIndex` | 列表页分页状态 | 从 `0` 开始，成功加载后自增 |
| `PageSize` | 列表页分页状态 | 刷新时设为 `20` |
| `IsLoadDetail` | 固定 | Legacy 列表设为 `true` |
| `Tag` | 固定 | Legacy 列表设为空字符串 |
| `travelformid` | URL Query | 出差单 ID，没有则为空 |
| `hotelType` | 搜索模型 | `Tmc` / `Normal` / `Agent` / 空 |
| `Passengers` | 当前预订乘客 | 乘客 AccountId 去重后用逗号拼接 |
| `staffCityCode` | localStorage | 员工城市编码 |
| `SearchKey` | 关键词 | 酒店名/地标/关键字文本 |
| `HotelId` | 关键词命中酒店时 | 关键词选择具体酒店后传酒店 ID，并清空 `SearchKey` |
| `Lat` / `Lng` | 关键词地理点或当前位置 | 经纬度搜索 |

### 3.3 分页与刷新

- 首次加载或下拉刷新：
  - `PageIndex = 0`
  - `PageSize = 20`
  - 清空本地列表 `hotelDayPrices`
  - 重置总数 `totalHotels`
  - 重新开启无限滚动
  - 滚动到顶部
- 加载更多：
  - 调用 `Home-List`
  - 若本页有数据，将 `PageIndex++`
  - 新数据追加到 `hotelDayPrices`
  - 返回数据小于 `PageSize` 时代表接近末页；Legacy 代码里未严格禁用 scroller，H5 实现时应明确处理「无更多」状态。
- 重置筛选：
  - 非保留查询条件刷新会重置 `HotelQueryEntity`、清空关键词、重置筛选 Tab 状态。
  - 修改日期会保留当前筛选条件刷新。

### 3.4 推荐/排序筛选

Tab：`推荐`

| 文案 | `Orderby` |
| --- | --- |
| 低价优先 | `PriceAsc` |
| 高价优先 | `PriceDesc` |
| 星级升序 | `CategoryAsc` |
| 星级倒序 | `CategoryDesc` |

选择排序项后：

- 设置唯一选中项。
- 写入 `hotelQuery.Orderby`。
- 关闭面板并刷新列表。

### 3.5 星级/价格筛选

Tab：`星级/价格`

- 星级，可多选，最多 3 个：
  - 一星、二星、三星、四星、五星
  - 写入查询字段：Legacy 将选中的 `stars` 映射到 `query.Stars`，最终请求里又赋给 `Categories`。
- 分类，可多选，最多 3 个：
  - 公寓、客栈、舒适、高档、豪华
  - 写入查询字段：`query.Categories`
- 价格，单选：
  - 150以下：`BeginPrice = 0`、`EndPrice = 150`
  - 150-300
  - 300-450
  - 450-600
  - 600以上：`EndPrice = 10000000`
- 自定义价格：
  - 双滑块，范围 0 到 1000+，步进 100。
  - 自定义价格优先覆盖价格档位。
- 操作：
  - 「不限」清空当前分组。
  - 「重置」重置所有星级/分类/价格。
  - 「确定」关闭面板并刷新列表。

### 3.6 位置区域筛选

Tab：`位置区域`

筛选源来自 `Condition-Gets` 的 `Geos`：

- 地铁：按线路二级分组，站点为三级项。
- 行政区
- 购物中心
- 商业中心
- 地标
- 医院
- 大学
- 演出场馆
- 市内景点
- 市外景点
- 兴趣点：TMC 集团/公司相关 `Group`、`Company`

选择规则：

- 同一时间只保留一个一级位置分组的选择；选择新分组会清空其它分组选择。
- 单组最多选 3 个。
- 确定后写入：
  - `query.Geos`
  - `query.searchGeoId`，优先取选中地铁站或第一个 Geo。
- 刷新列表时，Legacy 会把 `searchGeoId` 额外挂在 request 对象上，同时把 `Geos` 放入请求体。

### 3.7 综合筛选

Tab：`筛选`

筛选源来自 `Condition-Gets`：

- 品牌：
  - 热门品牌，取前 8 个。
  - 经济，取 `Tag = Economy` 前 20 个。
  - 舒适，取 `Tag = Comfort` 前 10 个。
  - 高端，取 `Tag = High` 前 10 个。
  - 豪华，取 `Tag = Luxury` 前 10 个。
- 主题：`Amenities.Tag = Theme`
- 服务：`Amenities.Tag = Service`
- 设施：`Amenities.Tag = Facility`

选择规则：

- 每个子分组最多选 3 个。
- 确定后写入：
  - `Themes`
  - `Brands`
  - `Services`
  - `Facilities`
- 若无任何综合筛选项选中，需要清空上述字段并刷新。

## 4. 酒店卡片

### 4.1 Legacy 展示字段

Legacy 卡片组件：`components/hotel-list-item_ryx/`

| 区域 | 字段/逻辑 |
| --- | --- |
| 图片 | `Hotel.FullFileName || Hotel.FileName`，失败时使用接口返回默认图 |
| 标签 | `Hotel.Tag` 包含 `GreenCloud` 显示「旅发酒店」；包含 `Tmc` 显示「协议酒店」；若同时包含 `GreenCloud` 和 `Tmc`，组件初始化时把 Tag 改为 `GreenCloud`，只显示旅发酒店 |
| 酒店名称 | `Hotel.Name` |
| 评分 | `Hotel.Grade >= 4` 时展示 `x分` |
| 星级 | 根据 `Hotel.Category` 生成星星；`>=5` 展示 5 星，小数展示半星 |
| 价格 | `Hotel["avgPrice"] ?? Hotel.AvgPrice`，文案 `￥xxx起` |
| 地址 | `Hotel.Address` |
| 距离 | 若 `Hotel.Distance` 存在，展示「距离{关键词}直线距离约{Distance}」 |

### 4.2 点击跳转

- 点击列表项进入酒店详情。
- 传参：
  - `hotelId`
  - `hotelprice`
- H5 当前跳转到 `/hotel/:hotelId`，并通过 Query 传递 `checkIn`、`checkOut`、`cityCode`、`cityName`、`minPrice`。

## 6. 迁移需求

### 6.1 迁移目标

将 Legacy `tmc-hotel-list_ryx` 的酒店列表查询、筛选排序、分页、卡片展示与详情跳转能力迁移到 `apps/h5`，使 H5 酒店列表页在业务行为上覆盖旧版融易行国内酒店列表页。

### 6.2 非目标范围

- 本文档聚焦「酒店列表页」。酒店搜索首页、关键词搜索页、酒店详情页、房型页、填写订单页只记录与列表页交互相关的上下游契约。
- 不照搬 Legacy Angular/Ionic 技术实现，仅迁移业务能力、字段语义和用户流程。

### 6.3 H5 视觉参考

酒店列表页迁移时，视觉与交互外壳参考 H5 机票列表，而不是照搬 Legacy Ionic 样式。

参考页面：

- 本地示例：`http://localhost:5173/flight/list?fromCode=BJS&toCode=SHA&fromName=北京&toName=上海&date=2026-06-29&fromAsAirport=false&toAsAirport=false`
- 设计规范：`docs/DESIGN.md` 的「H5 Travel List Chrome」
- 参考源码：
  - `apps/h5/src/pages/flight/FlightListPage.tsx`
  - `apps/h5/src/components/flight/FlightListHeader.tsx`
  - `apps/h5/src/components/flight/FlightListDateStrip.tsx`
  - `apps/h5/src/components/flight/FlightListToolbar.tsx`
  - `apps/h5/src/components/flight/FlightFilterSheet.tsx`
  - `apps/h5/src/components/flight/FlightSegmentCard.tsx`

酒店列表映射原则：

- 使用固定顶部渐变 Header 承载返回、当前查询摘要与右侧上下文动作。
- 日期/城市/关键词等高频修改入口使用紧凑 sticky strip 或 Header 下方紧凑控件，避免重新做大卡片搜索区。
- 酒店卡片沿用机票列表的密度、圆角、阴影、点击反馈和右侧价格锚点；图片作为酒店卡片左侧强识别区。
- 复杂筛选统一使用机票筛选 Sheet 的底部弹层结构：遮罩、65dvh 面板、分类左栏、内容右栏、重置/确定 footer。
- 过滤/排序控制优先使用一个主要控制面，避免顶部 Legacy 筛选 Tab 与底部工具栏同时存在造成重复。
- Legacy 功能必须保留，但视觉语言以 H5 travel list chrome 为准。

### 6.4 功能需求

#### FR-1 查询条

- 列表页顶部展示当前城市、入住/离店日期、晚数、关键词。
- 点击城市打开城市选择器；选择后更新 URL 查询参数并刷新列表。
- 点击日期打开日期选择器；确认后更新 URL 查询参数并刷新列表，保留筛选条件。
- 点击关键词进入关键词编辑/搜索能力；至少需要支持清空/修改关键词后刷新列表。

#### FR-2 列表加载与分页

- 首次进入列表页时按 URL 查询参数发起列表请求。
- 列表请求默认：`PageIndex = 0`、`PageSize = 20`、`IsLoadDetail = true`。
- 向下滚动触底加载下一页，新页数据追加到已有列表。
- 无更多数据时停止触底请求并展示轻量状态。
- 支持下拉刷新：重置 `PageIndex = 0`、清空列表、重新请求第一页。

#### FR-3 筛选元数据

- 根据当前 `CityCode` 调用 `TmcApiHotelUrl-Condition-Gets` 获取筛选元数据。
- 城市变更时重新获取条件。
- 条件加载失败时不阻塞基础列表展示；筛选面板展示失败/重试态，或禁用对应筛选入口。

#### FR-4 推荐/排序

- 点击「推荐」打开排序面板。
- 支持低价优先、高价优先、星级升序、星级倒序。
- 排序单选，选中后写入 `Orderby` 并刷新第一页。
- Tab 需要展示已筛选状态。

#### FR-5 星级/价格

- 点击「星级/价格」打开面板。
- 支持星级多选，最多 3 个。
- 支持分类多选，最多 3 个：公寓、客栈、舒适、高档、豪华。
- 支持价格单选：150 以下、150-300、300-450、450-600、600 以上。
- 支持自定义价格滑块，范围 0 到 1000+，步进 100；自定义价优先覆盖价格档。
- 支持分组「不限」、整体「重置」、「确定」。
- 确定后刷新第一页。

#### FR-6 位置区域

- 点击「区域位置」打开面板。
- 支持地铁、行政区、购物中心、商业中心、地标、医院、大学、演出场馆、市内景点、市外景点、兴趣点。
- 地铁支持线路/站点二级选择。
- 切换一级分组时清空其它分组选择。
- 单组最多选 3 个。
- 确定后写入 `Geos` 与 `searchGeoId` 并刷新第一页。

#### FR-7 综合筛选

- 点击「筛选」打开面板。
- 支持品牌、主题、服务、设施。
- 品牌按 Legacy 规则分组：热门品牌前 8 个，经济前 20 个，舒适前 10 个，高端前 10 个，豪华前 10 个。
- 每个子分组最多选 3 个。
- 确定后写入 `Brands`、`Themes`、`Services`、`Facilities` 并刷新第一页。

#### FR-8 酒店卡片

- 卡片需展示图片、标签、酒店名称、评分、星级、价格、地址、与关键词/定位点距离。
- 标签包括旅发酒店、协议酒店。
- `GreenCloud` 与 `Tmc` 同时存在时，按 Legacy 优先展示「旅发酒店」。
- 点击卡片进入详情页，并透传当前城市、日期、价格、酒店类型、出差单等必要上下文。

#### FR-9 随心住提示

- 若 H5 已接入 TMC 配置与自订/代理模式判断，需要按 Legacy 展示「超标随心住」提示。
- 提示点击后展示说明弹窗。
- 若当前迁移阶段暂未接入 TMC 配置，该功能可作为 P2，但需保留任务。

### 6.5 接口与类型需求

需要扩展 `packages/shared-types/src/hotel.ts`：

- `HotelListParams` 增加：`Orderby`、`BeginPrice`、`EndPrice`、`Categories`、`Geos`、`searchGeoId`、`Brands`、`Themes`、`Services`、`Facilities`、`TravelFormId`、`Passengers`、`StaffCityCode`、`Lat`、`Lng`、`HotelId`。
- `HotelListItem` 增加：`Grade`、`Distance`，并确保 `Tags` 能表达 `GreenCloud` 与 `Tmc`。
- 新增酒店条件类型：`HotelConditionResponse`、`HotelGeo`、`HotelBrand`、`HotelAmenity`。

需要扩展 `packages/api/src/apis/hotel.ts`：

- 新增 `getConditions(cityCode)`。
- `buildHotelListRequest` 支持上述筛选字段。
- `normalizeHotelListResponse` 映射 `Grade`、`Distance`、`GreenCloud` 标签。
- 默认 `PageSize` 与 Legacy 对齐为 `20`，除非产品明确要求保留 H5 当前 10 条。

### 6.6 状态与 URL

- 列表页需要将城市、日期、关键词等主查询条件放在 URL 中，便于返回详情后恢复。
- 筛选条件可优先放在组件状态中；如需支持分享/刷新恢复，再编码到 URL。
- 从详情页返回列表时，建议恢复已加载列表、当前筛选、当前滚动位置、当前分页。

## 7. 验收标准

- 从酒店搜索页进入列表页后，首屏可以正确展示城市、入住/离店日期、关键词、酒店列表。
- 列表接口请求包含 Legacy 必要字段：城市、日期、分页、酒店类型、关键词、乘客、出差单。
- 上拉触底能够加载下一页，且不会覆盖第一页数据。
- 下拉刷新能够重新加载第一页并回到顶部。
- 四个筛选 Tab 均可打开对应面板；点击遮罩或确定后面板关闭。
- 排序、星级/价格、位置区域、综合筛选任一条件变更后，列表从第一页重新请求。
- 已选择筛选条件时，对应 Tab 有选中态/提示态。
- 卡片展示协议酒店、旅发酒店、评分、星级、价格、地址、距离字段；字段缺失时页面不抖动、不展示错误文案。
- 点击卡片进入详情页，详情页可使用列表页传入的日期、城市、价格上下文正常加载。
- 接口失败时展示错误态与重试入口；空列表展示空态。
- Mock 模式覆盖基础列表、空态、筛选条件、筛选后列表、分页加载。
