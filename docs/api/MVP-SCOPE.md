# 产品 MVP 与页面矩阵的关系

> **更新**：2026-06-15  
> **主文档**：[PAGE-API-MATRIX.md](./PAGE-API-MATRIX.md)（29 页，按 Wave 迁移）

---

## 说明

本文档保留 **产品能力边界**；具体 Method 与迁移顺序以 **页面矩阵** 为准。

**需求**：App 内申请出差单 → 订酒店/机票/火车 → 支付完成。

---

## 不做的模块

| 模块 | 说明 |
|------|------|
| CRM | 客户销售 |
| MMS | 商城 |
| 游客态 TmcTourist* | 散客 |
| 国际机/酒 | 除非 PRD 追加 |
| BPM 全套 | 仅取出差申请子集 |

---

## MVP 能力 → Wave 映射

| 产品能力 | 对应 Wave | 矩阵页面 |
|----------|-----------|----------|
| 登录 | 1 | 手机/密码登录 |
| 首页工作台 | 2 | Tab + home |
| 申请出差单 | 7 | `/travel/apply`（抓包） |
| 选择出差单 | 7 | 预订内嵌 ✅ |
| 订酒店 | 3 | 搜索→列表→详情→填单→支付 |
| 订机票 | 5 | 搜索→列表→填单 |
| 订火车 | 6 | 搜索→车次→填单 |
| 订单/支付 | 3+4 | 结果页 + 订单中心 |
| 我的/待出行 | 2+4 | Tab + 订单列表 |

## 出差单：两件不同的事

|------|----------|------|
