# 核心业务逻辑

## 产品定位

**企业差旅管理（TMC）移动客户端 — 融易行**

面向企业员工的因公出行场景，提供机票、酒店、火车、用车等预订能力，并对接差旅申请单与审批流程。

## 主流程概览

```
登录 → 选择公司/Agent → 首页选产品 → 搜索 → 选乘客/填信息 → 下单 → 支付 → 订单管理
                ↑
         差旅申请单 / 审批（合规校验）
```

## 核心服务：`TmcService`

路径：`src/app/tmc/tmc.service.ts`

主要职责：

| 能力         | 说明                                                  |
| ------------ | ----------------------------------------------------- |
| 基础资源缓存 | 国内/国际机场、火车站（带 `LastUpdateTime` 增量更新） |
| 多租户       | 当前 TMC 公司、Agent 切换与缓存                       |
| 工作台       | Workbench 数据按 HR 维度加载                          |
| 差旅申请单   | TravelForm 关联、合规提示                             |
| 乘客证件     | 与 `MemberService` 协作加载员工证件                   |
| 支付         | 与 `PayService` 协作                                  |

## 业务域详解

### 1. 身份认证

- 密码 / 手机号 / 设备登录
- 登录成功后保存 Ticket，再请求 WebSocket URL
- 未登录或 Ticket 失效跳转 `login`

### 2. 多租户

- 用户可绑定多个 TMC 公司
- `tmc-switch-company` 切换当前操作主体
- Agent 模式有独立 Guard（`agent.guard.ts`）

### 3. 差旅预订

| 产品     | 模块                      | 典型流程                               |
| -------- | ------------------------- | -------------------------------------- |
| 机票     | `tmc-flight`              | 选城市 → 搜航班 → 选舱 → 填乘客 → 下单 |
| 酒店     | `tmc-hotel`               | 选城市 → 选日期 → 列表 → 详情 → 预订   |
| 国际酒店 | `tmc-international-hotel` | 同上，国际资源                         |
| 火车     | `tmc-train`               | 选站 → 搜车次 → 选座 → 下单            |
| 用车     | `tmc-car`                 | 租车 / 开放平台用车                    |

### 4. 差旅合规

- 预订前可关联 **差旅申请单**（TravelForm）
- 超标或需审批时走 `tmc-approval-task`
- 政策校验接口（如 `HOME_POLICY`）返回是否违规

### 5. 乘客管理

- 从企业员工列表选择乘客
- 校验证件类型与号码（`member/pipe/credential.pipe`）
- 支持多人预订

### 6. 订单与支付

1. `tmc-confirm-information` — 确认订单信息
2. 创建订单 → 唤起支付
3. `tmc-checkout-success` — 支付成功页
4. `order/` — 历史订单列表与详情

## 底部 Tab 信息架构

企业主线（`isTmcHome`）：

| Tab    | 旧路由模块         | 说明                       |
| ------ | ------------------ | -------------------------- |
| 首页   | `tab-tmc-home_ryx` | 产品入口、Banner、快捷预订 |
| 待出行 | `tab-tmc-trip_ryx` | 行程列表                   |
| 我的   | `tab-tmc-my_ryx`   | 个人中心、设置             |

另有游客首页（`isPublicHome`）、BPM 壳（`isBpmHome`）等变体，企业主线以 `isTmcHome` 为准。

## 基础数据 API（Resource 类）

| Method                                        | 用途            | 调用方（Service）       |
| --------------------------------------------- | --------------- | ----------------------- |
| `TmcApiHomeUrl-Resource-Airport`              | 国内机场 / 城市 | `TmcService`            |
| `TmcApiHomeUrl-Resource-InternationalAirport` | 国际机场        | `TmcService`            |
| `TmcApiHomeUrl-Resource-TrainStation`         | 火车站          | `TmcService`            |
| `TmcApiHomeUrl-Resource-DomesticHotelCity`    | 国内酒店城市    | `TmcHotelRyxService` 等 |

客户端拉取后写入 Ionic Storage，按 `LastUpdateTime` 增量更新。酒店城市**不由** `TmcService` 加载，详见 [酒店模块.md](./酒店模块.md)。
