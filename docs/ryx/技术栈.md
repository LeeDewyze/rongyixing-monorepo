# 技术栈

## 概览

| 类别        | 技术                                                                                                                              |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------- |
| 语言        | **TypeScript**                                                                                                                    |
| 前端框架    | **Angular 12** + **Ionic 6**                                                                                                      |
| 移动端      | **Capacitor 3**（Android / iOS 混合 App）                                                                                         |
| 构建工具    | **Nx Monorepo**（位于 `beeantmobile-main` 工作区）                                                                                |
| 网络请求    | 三层：**ApiService**（facade）→ **CoreApiService**（Sign/Urls/Token）→ **HttpClient**；Beeant Proxy（`Method` + `Data` RPC 风格） |
| 状态 / 缓存 | **RxJS** + **Ionic Storage**（本地缓存，无客户端数据库）                                                                          |
| 测试        | Jest、Karma                                                                                                                       |
| 国际化      | zh-cn / en JSON 资源文件                                                                                                          |

## API 客户端分层

业务 Service 调用 `ApiService`，实际逻辑在 `CoreApiService`（`projects/core/src/services/api/core-api.service.ts`，~880 行）：

| 层                 | 文件                  | 职责                                                                                                  |
| ------------------ | --------------------- | ----------------------------------------------------------------------------------------------------- |
| **ApiService**     | `api.service.ts`      | 薄 facade：`send`、`getPromiseData` 等委托给 CoreApiService                                           |
| **CoreApiService** | `core-api.service.ts` | Method → Url 解析（`appConfig.Urls` 前缀映射）、Sign 计算、Token 注入、超时/重试、WebSocket、自动重登 |
| **HttpClient**     | Angular               | 实际 HTTP POST（`application/x-www-form-urlencoded`）                                                 |

`ApiService` 仅 ~100 行；理解 legacy 网络层须以 **CoreApiService** 为准。详见 [架构.md](./架构.md#api-通信模式)。

## 路由与打包

- 路由表广泛使用 `loadChildren` 动态 import 写法
- 但 `TmcRoutingModule` / `RyxRoutingModule` 在 `NgModule.imports` 中 **eager import** 了 `TmcFlightModule`、`TmcTrainModule`、`TmcHotelRyxModule` 等主模块
- 实际 **code splitting 效果有限**，主 bundle 仍包含大量 TMC 业务代码；新 H5 可据此制定更激进的路由级分包策略

## 常用依赖

- **moment.js** — 日期处理
- **cropperjs** — 图片裁剪
- **vConsole** — 移动端调试
- **微信 SDK** — 分享、登录等
- **厂商推送** — 小米 / 华为 / OPPO / vivo

## 数据存储说明

- **无客户端数据库**（无 SQLite / IndexedDB ORM）
- 业务数据持久化在**后端服务**（`app.rongtrip.cn` 生产 / `app.rtesp.cn` 测试）
- 客户端通过 **Ionic Storage** 做 Ticket、机场/车站/城市列表等缓存
