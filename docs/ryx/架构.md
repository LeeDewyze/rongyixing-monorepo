# 整体架构

## 架构模式

**Nx 多项目 + 功能模块化分层架构**（非 MVC、非前端微服务）

```
┌─────────────────────────────────────┐
│  ryx App（Ionic Pages / Modules）    │  ← 业务 UI 层
├─────────────────────────────────────┤
│  ryx Services（TmcService 等）       │  ← 业务逻辑层
├─────────────────────────────────────┤
│  projects/core（CoreApiService 等）  │  ← 共享基础设施层
├─────────────────────────────────────┤
│  projects/models（DTO / Entity）     │  ← 数据模型层
└─────────────────────────────────────┘
           ↓ HTTP Proxy
      后端 TMC API（/Home/Proxy）
```

## 运行形态

| 形态           | 说明                                                        |
| -------------- | ----------------------------------------------------------- |
| **Hybrid App** | 同一套 Angular 代码，H5 与 Capacitor 原生壳共用             |
| **H5**         | 浏览器 / WebView 访问，路由可用 Hash 模式                   |
| **Native**     | Capacitor 打包 Android / iOS，`appId: com.ronglvonline.app` |

## 模块组织

- **Feature Module 拆分**：按业务域（机票、酒店、火车等）独立 `NgModule`
- **路由**：子模块路由表含 `loadChildren`，但 `TmcRoutingModule` 等对主模块做 **eager import**，bundle 分割有限（详见 [技术栈.md](./技术栈.md#路由与打包)）
- **路由守卫**：`guards/` 下 TMC、Agent 等鉴权
- **共享能力下沉到 core**：Sign、Urls 映射、Identity、Loading、Storage 不重复实现

## API 通信模式

业务 Service 调用 `ApiService`（facade），核心逻辑在 **`CoreApiService`**（`core-api.service.ts`）：

```typescript
// 业务层典型用法
apiService.getPromiseData<T>(req); // req.Method = "TmcApiFlightUrl-Home-Index"

// ApiService 内部委托
coreApiService.getPromiseData(req, onRemoveIdentity, onApiTimeout);
```

### Urls 前缀映射（Method → 微服务基址）

启动时拉取 `/Home/Setting`（`loadApiConfig`）得到 `appConfig.Urls`。请求时 `CoreApiService.getUrl()` 按 Method 前缀解析目标 URL：

```
Method: "TmcApiFlightUrl-Home-Index"
  → split("-") → ["TmcApiFlightUrl", "Home", "Index"]
  → appConfig.Urls["TmcApiFlightUrl"]
  → "https://flight-api-tmc.rongtrip.cn"
  → 最终 URL: https://flight-api-tmc.rongtrip.cn/Home/Index
```

未匹配到 `Urls` 前缀时，fallback 为 `CoreHelper.getApiUrl() + "/Home/Proxy"`。

同一套 `Method` 字符串因此可路由到不同微服务（机票、酒店、火车等），无需业务层硬编码域名。

### 典型请求流程

1. 启动时拉取 `/Home/Setting` 获取 Token 与 **Urls 映射**
2. 登录后保存 Ticket，后续请求由 CoreApiService 拼装 **Sign** 并注入 Token
3. `getUrl(req)` 将 Method 解析为微服务 URL + 路径段
4. HttpClient POST，响应包裹在 Beeant 标准 `IResponse<T>` 信封中
5. Token 失效时触发 `onRemoveIdentity` 跳转登录；支持自动重登（`tryAutoLogin`）
